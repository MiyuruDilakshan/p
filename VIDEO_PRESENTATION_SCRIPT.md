### SPEECH (Simple & Clear)
"Good morning everyone. We are presenting our project called **Roomio**.

Roomio is a web application that helps customers and designers visualize furniture in rooms before buying. 

**The problem:** People often buy furniture but realize it doesn't fit their room or looks bad with their decoration. They waste money and time.

**Our solution:** Roomio lets users create their room in the application, add furniture, and see exactly how it looks in both 2D top-down view and realistic 3D view. This helps them make better buying decisions.

Our team built this application to solve this real problem. Let me hand over to our next team member who will explain the design."

### SPEECH (Simple & Clear)
"Hello, I'm explaining the design of Roomio. We followed HCI - Human Computer Interaction - principles to make the app easy to use.

**First, we created low-fidelity mockups.** These are simple sketches showing the basic layout. We did this to test our ideas quickly.

*[SHOW Slide 2 - Point to wireframes]*

Then we created high-fidelity designs in Figma. These look like the real application with colors and details.

*[SHOW Slide 3 - Point to different pages]*

**The main pages are:**

1. **Landing page** - where users first see the app and learn about it
2. **Dashboard** - where users see their saved room designs
3. **Room Designer** - the main tool where users create and edit rooms
4. **Furniture Library** - shows all available furniture items

**HCI Principles we used:**

- **Consistency**: Same buttons and colors everywhere so users don't get confused
- **Visibility**: System tells users what happened. Example: 'Design saved successfully'
- **Error Prevention**: Clear messages before deleting anything
- **Efficiency**: Users can complete tasks quickly with few clicks

We designed everything simple and intuitive so even first-time users understand it quickly.

Now, let me hand over to our tech team for the live demo of the actual application."

## 👤 MEMBER 5 (Girl) - CODE EXPLANATION
**Duration: 1.5 minutes (7:30 - 9:00)**

### SCREEN SETUP
- GitHub repository open in browser
- PowerPoint Slide 5: "MERN Stack Architecture"
- Show folder structure with these components

### SPEECH (Simple & Clear)
"Now I explain the technology and code behind Roomio.

We built this using the **MERN Stack**. What is MERN?

*[SHOW: Architecture diagram]*

**M** - MongoDB: Database where we store room designs and user information
**E** - Express: Backend framework that handles requests
**R** - React: Frontend framework for the user interface
**N** - Node.js: JavaScript runtime for the server

**Let me show the code structure:**

*[OPEN: GitHub and show folder structure]*

In the **server folder**, we have:
- `/routes` - API endpoints for login, saving designs, getting furniture
- `/models` - Database schemas for User, Design, and Furniture
- `/controllers` - Logic for handling requests

*[SHOW: server.js file briefly]*

In the **client folder**, we have:
- `/pages` - Web pages like Dashboard, Room Designer
- `/components` - Reusable parts like FurnitureLibrary, Room2DPlan
- `/services` - Code that talks to the backend API

*[SHOW: client folder structure]*

**Key components:**

1. **RoomDesignerPage.jsx** - Main room design tool
2. **DesignerCanvas.jsx** - 3D room visualization using Three.js
3. **Room2DPlan.jsx** - 2D top-down floor plan
4. **FurnitureLibrary.jsx** - List of all furniture items

*[Point to each file briefly]*

**Database Structure:**

When user saves a design, we store:
- Room dimensions (length, width, height)
- Wall colors and floor materials  
- List of furniture items with positions and scales
- User ID (so each user sees only their designs)

*[SHOW: Design model briefly]*

The application uses **protected routes** - users cannot access other people's designs because of authentication.

Everything is hosted locally using **MongoDB locally** or **MongoDB Atlas** for cloud storage.

This is a professional, clean architecture that follows best practices for web development.

Now, let me pass to our next member to discuss user testing."
